# 🎭 演示模式

## 什么是演示模式？

演示模式允许您**无需安装 MongoDB** 就能运行和体验本项目的所有功能。系统会使用预设的模拟数据来展示产品对比功能。

## ✨ 特点

- ✅ **无需数据库** - 不需要安装 MongoDB 或 Docker
- ✅ **完整功能** - 可以浏览、查看和对比产品
- ✅ **快速体验** - 立即启动，无需额外配置
- ⚠️ **只读模式** - 无法添加、编辑或删除数据

## 🚀 启用演示模式

### 方法一：修改 .env 文件（当前已启用）

编辑 `.env` 文件，设置：

```env
DEMO_MODE=true
```

### 方法二：环境变量

```bash
DEMO_MODE=true npm run dev
```

## 🔄 切换到完整模式

要使用完整功能（包括数据管理），您需要：

1. 安装并启动 MongoDB：

   ```bash
   # 使用 Docker
   docker run -d -p 27017:27017 --name mongodb mongo
   
   # 或使用 MongoDB Atlas（免费云数据库）
   # 查看 docs/mongodb-atlas-setup.md
   ```

2. 修改 `.env` 文件：

   ```env
   DEMO_MODE=false
   MONGODB_URI=mongodb://localhost:27017/robot-comparison
   # 或使用 Atlas 连接字符串
   ```

3. 填充数据：

   ```bash
   npm run seed
   ```

4. 重启应用：

   ```bash
   npm run dev
   ```

## 📝 注意事项

1. **数据不会保存** - 演示模式下的所有操作都不会被持久化
2. **管理后台受限** - 后台页面会显示，但修改操作不会生效
3. **重启恢复** - 每次重启应用，数据都会重置为初始状态

## 🔧 故障排除

### 页面显示"暂无产品数据"

确认 `.env` 文件中设置了：
```env
DEMO_MODE=true
```

### API 返回空数据

1. 检查 `.env` 文件是否存在
2. 确认 `DEMO_MODE=true` 设置正确
3. 重启开发服务器

### 对比功能不工作

这是正常的，演示数据已经预设好，请：
1. 访问产品列表页
2. 选择任意2-4个产品
3. 点击"对比选中产品"
